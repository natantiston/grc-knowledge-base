# Chapter 09 – Payment Card Industry Data Security Standard (PCI DSS) 4.0

## Overview

The **Payment Card Industry Data Security Standard (PCI DSS) 4.0** is the global security standard for organizations that store, process, or transmit payment card data. Developed by the **PCI Security Standards Council (PCI SSC)**, PCI DSS provides a comprehensive framework of security requirements designed to protect cardholder data, reduce payment fraud, and strengthen cybersecurity across the payment ecosystem.

PCI DSS applies to merchants, financial institutions, payment service providers, processors, software vendors, cloud service providers, and any organization that handles payment card information. Compliance requires implementing technical, administrative, and operational controls that secure the entire Cardholder Data Environment (CDE).

This chapter introduces PCI DSS 4.0, explains its objectives and scope, and provides an in-depth examination of each of the twelve PCI DSS requirements. Throughout the chapter, readers will learn how organizations implement enterprise security controls, perform risk-based compliance activities, protect payment data, monitor security operations, and establish governance programs that support long-term compliance.

By the end of this chapter, you will understand how PCI DSS integrates with enterprise cybersecurity frameworks such as ISO/IEC 27001, NIST Cybersecurity Framework (CSF), COBIT, and Governance, Risk, and Compliance (GRC) programs.

---

## Learning Objectives

After completing this chapter, you will be able to:

- Understand the purpose, scope, and structure of PCI DSS 4.0.
- Explain the responsibilities of organizations that store, process, or transmit payment card data.
- Understand the concept of the Cardholder Data Environment (CDE).
- Identify and explain each of the twelve PCI DSS security requirements.
- Learn how PCI DSS supports confidentiality, integrity, and availability of payment information.
- Understand how technical, administrative, and physical security controls work together to protect cardholder data.
- Perform risk-based assessments aligned with PCI DSS requirements.
- Understand enterprise implementation of PCI DSS governance and compliance programs.
- Prepare organizations for PCI DSS assessments conducted by Qualified Security Assessors (QSAs).
- Integrate PCI DSS with enterprise Governance, Risk, and Compliance (GRC) and cybersecurity programs.

# Lessons

## Lesson 9.1 – Introduction to PCI DSS 4.0

Understand the purpose, history, scope, structure, and objectives of PCI DSS 4.0, including the Cardholder Data Environment (CDE), applicability, compliance lifecycle, and the overall security philosophy behind protecting payment card information.

---

## Lesson 9.2 – Requirement 1: Install and Maintain Network Security Controls

Learn how organizations protect payment environments using firewalls, network segmentation, secure network architecture, boundary protection, and network security management to prevent unauthorized access.

---

## Lesson 9.3 – Requirement 2: Apply Secure Configurations to All System Components

Explore secure configuration management, system hardening, baseline security configurations, configuration standards, and secure deployment practices that reduce system vulnerabilities.

---

## Lesson 9.4 – Requirement 3: Protect Stored Account Data

Learn how organizations protect stored payment account data using data classification, retention policies, encryption, tokenization, truncation, masking, and cryptographic key management.

---

## Lesson 9.5 – Requirement 4: Protect Cardholder Data with Strong Cryptography During Transmission over Open, Public Networks

Understand how strong cryptography, TLS, VPNs, digital certificates, secure APIs, wireless security, and certificate management protect payment data during transmission across public networks.

---

## Lesson 9.6 – Requirement 5: Protect All Systems and Networks from Malicious Software

Learn how organizations deploy anti-malware solutions, endpoint detection and response (EDR), malware prevention, threat intelligence, endpoint monitoring, and secure endpoint management to defend against malicious software.

---

## Lesson 9.7 – Requirement 6: Develop and Maintain Secure Systems and Software

Explore secure software development, vulnerability management, patch management, DevSecOps, secure coding practices, software testing, and application security throughout the software development lifecycle.

---

## Lesson 9.8 – Requirement 7: Restrict Access to System Components and Cardholder Data by Business Need to Know

Understand least privilege, role-based access control (RBAC), access provisioning, segregation of duties, privileged access management, and access governance to minimize unauthorized access.

---

## Lesson 9.9 – Requirement 8: Identify Users and Authenticate Access to System Components

Learn how strong authentication mechanisms—including multi-factor authentication (MFA), identity lifecycle management, password policies, authentication protocols, and privileged identity management—protect access to payment systems.

---

## Lesson 9.10 – Requirement 9: Restrict Physical Access to Cardholder Data

Explore physical security controls, facility access management, visitor management, media protection, environmental security, surveillance systems, and secure disposal of sensitive media.

---

## Lesson 9.11 – Requirement 10: Log and Monitor All Access to System Components and Cardholder Data

Learn how logging, audit trails, Security Information and Event Management (SIEM), continuous monitoring, log protection, and security analytics enable organizations to detect suspicious activities and support forensic investigations.

---

## Lesson 9.12 – Requirement 11: Test Security of Systems and Networks Regularly

Understand vulnerability management, vulnerability scanning, penetration testing, wireless security testing, file integrity monitoring (FIM), intrusion detection, security validation, remediation, and continuous testing of enterprise security controls.

---

## Lesson 9.13 – Requirement 12: Support Information Security with Organizational Policies and Programs

Learn how governance, security policies, enterprise risk management, security awareness, third-party risk management, executive oversight, compliance monitoring, and continuous improvement establish a sustainable enterprise information security program.

---

## Lesson 9.14 – Enterprise Integration and Governance for PCI DSS 4.0

Bring together all PCI DSS requirements into an integrated enterprise compliance program by exploring governance frameworks, performance measurement, continuous compliance, organizational accountability, executive reporting, audit readiness, and long-term operational maturity.
